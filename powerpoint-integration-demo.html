How to wire a PowerPoint up to the web

Below are the five most common patterns — from the simplest “drop it on a page” embed, all the way to fully data-driven decks generated from your CMS or database. Pick whichever one fits.

1 Embed in any website (Office Online viewer)

The fastest option. Host the .pptx on a public URL (GitHub Pages, Azure Blob, S3, your own site) and embed Microsoft’s free Office Online viewer in an <iframe>. 

<iframe
  src="https://view.officeapps.live.com/op/embed.aspx?src=https%3A%2F%2Fyourdomain.com%2Fdecks%2FSalesReview.pptx"
  width="100%" height="600" frameborder="0">
</iframe>
The file must be reachable over HTTPS with no auth required. For private files, use the SharePoint/OneDrive embed (method 2).

2 Embed from SharePoint or OneDrive

Upload the deck to a SharePoint document library or OneDrive folder, open it, then choose File → Share → Embed. SharePoint will generate a ready-to-paste iframe scoped to the file’s permissions (only people with access can view it). 

<!-- Auto-generated by SharePoint → File → Embed -->
<iframe
  src="https://contoso.sharepoint.com/sites/Sales/_layouts/15/embed.aspx?UniqueId=ABC-123"
  width="962" height="565"
  frameborder="0" scrolling="no" allowfullscreen>
</iframe>

Inside SharePoint itself, drop the File Viewer web part on a page and point it at the .pptx — you’ll get the same live deck with slide navigation, full-screen, and PowerPoint Live in Teams.

3 Bind web data into the template (placeholder swap)

The XML template uses double-brace tokens like {{$12.4M}} and {{104%}}. The pattern is the same whether you do this in the browser (PptxGenJS), Node.js (docxtemplater + pptx module), or Python (python-pptx): load the file, walk every text run, and replace the tokens with values from your data source. 

# Python example using python-pptx
from pptx import Presentation

data = {
    "REVENUE":         "$12.4M",
    "REVENUE_DELTA":   "+8.2% YoY",
    "PLAN_ATTAINMENT": "104%",
    "PIPELINE":        "$18.1M",
    "WIN_RATE":        "36%",
}

prs = Presentation("SalesReview-Template.pptx")
for slide in prs.slides:
    for shape in slide.shapes:
        if not shape.has_text_frame: continue
        for para in shape.text_frame.paragraphs:
            for run in para.runs:
                for k, v in data.items():
                    run.text = run.text.replace(f"{{{{{k}}}}}", v)

prs.save("SalesReview-Q2-2026.pptx")

4 Generate a .pptx straight from the browser (PptxGenJS)

For dashboards where users click a button and download a ready-made deck, you don’t even need a template file — PptxGenJS builds a fully native PowerPoint file in JavaScript. The “Generate a .pptx from this page” button above runs exactly this code: 

const pptx = new PptxGenJS();
pptx.layout = "LAYOUT_WIDE";

const slide = pptx.addSlide();
slide.addText("BUSINESS REVIEW", { x:0.5, y:0.4, fontSize:10, bold:true, color:"005EB8" });
slide.addText(sampleData.title, { x:0.5, y:0.7, fontSize:28, bold:true });

// KPI tiles
sampleData.kpis.forEach((k, i) => {
  const col = i % 2, row = Math.floor(i/2);
  slide.addText(
    [{ text:k.label+"\n", options:{ fontSize:9,  bold:true, color:"5C6670" } },
     { text:k.value+"\n", options:{ fontSize:22, bold:true, color:"1B1F24" } },
     { text:k.delta,        options:{ fontSize:10, bold:true, color:"2E7D32" } }],
    { x:0.5+col*1.85, y:2.6+row*1.15, w:1.75, h:1, fill:{color:"FFFFFF"}, line:{color:"C8D3E0"} }
  );
});

// Native PowerPoint chart from the same sample data
slide.addChart(pptx.ChartType.bar, [
  { name:"Plan",   labels:sampleData.chart.labels, values:sampleData.chart.plan   },
  { name:"Actual", labels:sampleData.chart.labels, values:sampleData.chart.actual },
], { x:5, y:2.2, w:7.5, h:3, chartColors:["C8D3E0","005EB8"] });

pptx.writeFile({ fileName:"SalesReview.pptx" });

5 Push the generated deck into SharePoint / Teams (Microsoft Graph)

Once you have a .pptx (from method 3 or 4), drop it straight into a SharePoint library or a Teams channel’s Files tab with one PUT call to the Microsoft Graph API. Teams & Outlook will then render an inline preview automatically. 

// Upload a file < 4 MB to a SharePoint document library
await fetch(
  `https://graph.microsoft.com/v1.0/sites/${siteId}/drives/${driveId}` +
  `/root:/Sales%20Reviews/SalesReview-Q2-2026.pptx:/content`,
  {
    method: "PUT",
    headers: {
      "Authorization": `Bearer ${accessToken}`,
      "Content-Type":  "application/vnd.openxmlformats-officedocument.presentationml.presentation",
    },
    body: pptxBlob,   // Blob returned by PptxGenJS write({ outputType:"blob" })
  }
);
For files larger than 4 MB use an createUploadSession resumable upload. The same Graph endpoint works for OneDrive (/me/drive) and Teams channels (the channel’s Files tab is just a folder in the team’s SharePoint site).

✓ What this demo proves

  • The same sampleData object drives the on-page KPIs, the live Chart.js chart, the data table, and the generated .pptx — one source of truth.
  • The HTML view can live on your website; the .pptx version can live in SharePoint — both stay in sync because both are rendered from your data.
  • You can swap the in-page sampleData for a fetch() against your real API and the rest of the page (and the generated deck) will just work.
