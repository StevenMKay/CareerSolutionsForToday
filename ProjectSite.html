Quick Start Guide

Welcome to the Cross-Impact, Roles, and Resource Guide! This tool helps you track projects through three phases:

1. Strategy → 2. Readiness → 3. Issue Resolution
  • Create an Item: Click "Add Item" on the Items tab. Fill in the Basic Information and choose an Intake Type.
  • Change Phase: Use the "Intake Type" dropdown in the modal to move items between phases.
  • Track Progress: The chevron at the top shows the current phase; checkmarks indicate completed phases.
  • Assign Owners: Owner dropdowns are populated from the Org Chart. Update the org chart to add new people.

Tab Reference

Items

Main workspace. Create, view, and edit projects/issues. Filter by phase using the cycle button. Click any row to open the edit modal.

Action Steps

Breakdown of tasks for each item. Linked to parent items for tracking completion status.

Book of Work

High-level view of all work items across lines of business (LOB). Useful for portfolio management.

Org Chart

Three team org charts (Strategy, Readiness, Issue). Names from these charts populate owner dropdowns in forms.

Data

Analytics dashboard. View distribution of items by type, trends over time, and phase duration metrics.

SP Tools

SharePoint helper for developers. Discovers column names and types for API integration.

Phase & Section Visibility

When editing an item, the sections shown depend on the current Intake Type. This ensures users only see relevant fields:

Intake Type Visible Sections Team Color
Project Initiation Basic Info, Project Initiation, Attachments, Notes Purple border
Readiness Review Basic Info, Project Initiation, Impact Assessment, Attachments, Notes Orange border
Issue Resolution Basic Info, Project Initiation, Impact Assessment, Issue Resolution, Attachments, Notes Red border

TIP The colored left border on each section indicates which team is responsible for that section.

Customization Guide (For Developers)

Adding New LOB Options

To add new Line of Business options, find the itemLob select element in the Basic Information section and add new <option> tags.

Modifying Phase Colors

Phase colors are defined in CSS. Search for these class patterns:

  • .strategy - Purple (#8e44ad)
  • .readiness - Orange (#f39c12)
  • .issue - Red (#e74c3c)

Adding New Form Sections

  1. Add HTML section in the modal body (copy existing section pattern)
  2. Add section ID to sectionRules object in updateSectionsVisibility()
  3. Add jump button in .jump-nav
  4. Add save/load logic in saveItem() and openEditModal()

Connecting to SharePoint

Replace placeholder functions with actual SharePoint REST API calls:

  • getSPData(listName) - Replace with _api/web/lists/getbytitle('...')/items
  • addSPListItem(listName, data) - POST to items endpoint
  • updateSPListItem(listName, id, data) - MERGE to item endpoint

NOTE Use the "SP Tools" tab to discover actual SharePoint column names for your environment.

Troubleshooting

  • Owner dropdowns empty: Check the Org Chart tab. Names are pulled from there. Add team members to see them in dropdowns.
  • Sections not showing: Verify the Intake Type is set correctly. Different phases show different sections.
  • Data not saving: Check browser localStorage permissions. Some browsers block localStorage in private/incognito mode.
  • Charts not loading: Ensure Chart.js CDN is accessible. Check browser console for errors.
  • Phase history not updating: Phase history only adds new phases, it doesn't remove them. This is intentional for audit trail.
